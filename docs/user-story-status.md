# RCH Rust Canonical User Story Status

This is the single canonical user-story ledger for release-candidate feature
audit work. It tracks operator-facing features, expected behavior, current
evidence, test status, and errors found during manual or automated story runs.

Last updated: 2026-06-23

## Status Values

- `Inventory`: Story identified from code/docs, expected behavior drafted, test pass not started.
- `Covered`: Automated tests or prior release evidence cover the expected behavior.
- `Manual pass`: A human/browser/live-device pass verified the behavior.
- `Needs manual test`: Automated coverage exists or source behavior is known, but the user workflow still needs an end-to-end manual pass.
- `Error found`: A logistical, UX, runtime, or behavioral error is recorded and needs a fix.
- `Retest needed`: A fix landed; the story must be exercised again.
- `Blocked`: A dependency, device, credential, or external service prevents meaningful testing.
